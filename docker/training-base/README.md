# hkask-training-base

The ONE generic minimal Docker image for ALL hKask training jobs.

**Image size: ~130MB.** Contains nothing but the bones: `python:3.11-slim` + `bash` + `curl` + `git` + `build-essential`. No axolotl, no TRL, no Ludwig, no PyTorch, no CUDA toolkit, no pre-baked anything.

## Design

This image replaces the previous per-harness Docker images (`axolotl-lora-trainer`, `trl-lora-trainer`). Instead of baking harness-specific packages into separate images, this single image receives an install script at pod startup that pip-installs whatever the selected harness needs.

**Why this approach:**
- Previous attempts with pre-baked images produced 22-31GB images that took hours to push and caused OOM during pip install (Lesson 11).
- Per-harness images multiply maintenance — each needs its own build, push, and version tracking.
- The install script is generated by Rust at submit time, so it always matches the exact harness version and training config — no image rebuild needed when TRL or Ludwig releases a new version.

## How it works

1. Rust (`RunpodHost::submit()`) renders the training config (YAML or Python) AND generates an install script as a bash script.
2. The install script is passed to the pod as the `HKASK_INSTALL_SCRIPT` env var.
3. The pod's entrypoint writes the script to `/workspace/install_and_train.sh` and executes it.
4. The install script:
   - Sets up environment variables (HF_HOME, PIP_CACHE_DIR, etc.)
   - pip-installs the harness-specific packages with pinned versions
   - Writes the rendered config to `/workspace/config.yml` or `/workspace/train.py`
   - Runs the training command (`axolotl train`, `python train.py`, `ludwig train`)
   - Uploads the adapter to HuggingFace
   - Writes a completion manifest
   - `exec sleep infinity` for SSH debugging

## Install script generation

The install script is generated by `RunpodHost::generate_install_script()` in Rust, based on:
- The selected harness (axolotl, TRL, Ludwig)
- The selected trainer (SFT, DPO, KTO, ORPO, Reward, GRPO)
- The rendered training config (already generated by the harness's `render_config()`)
- The HuggingFace model repository and dataset info

The lora-training skill's G6 gate determines which harness to use. The install script is the bridge between the skill's recommendation and the pod's execution — it installs exactly what the harness needs, no more, no less.

## Build & push

```bash
cd docker/training-base
podman build -t docker.io/mdzaxo/hkask-training-base:latest .
podman push docker.io/mdzaxo/hkask-training-base:latest
```

Build completes in seconds. Image is ~130MB uncompressed, ~44MB compressed.

## What is NOT in this image

- No CUDA devel base image — RunPod pods provide the host CUDA driver.
- No PyTorch — pip-installed at pod startup as a harness dependency.
- No axolotl, TRL, or Ludwig — pip-installed at pod startup.
- No pre-baked model weights or datasets — pulled from HuggingFace at runtime.
- No Python config generator — the config is rendered in Rust and passed as an env var.
- No per-harness images — one image serves all harnesses.
