## Key Decisions and Rationale

1. **`*Kind` phantom types kept alive.** They showed 0 external usage but are sealed phantom type tags for `Id<T>`. Deleting breaks type-safe IDs. Correct Rust design.

2. **`CalibratedThresholds` as Arc<RwLock> layer.** `set_points` is immutable (tick takes `&self`). Only 3 calibratable fields in separate `Arc<RwLock>` that tick reads and calibrator writes.

3. **SeamWatcher in tick, not background task.** `check_drift()` needs `&mut self` + `&CnsRuntime` + `&NuEventSink` (all available in tick). Background task can't access `event_sink` (set after builder).

4. **Calibrators NOT consolidated.** Different pipeline stages: `CalibratedEnergyEstimator` = "how much gas will this cost?" (pre-exec, gas-denominated); `WalletGasCalibrator` = "how much gas does one rJoule buy?" (post-settlement, gas→rJoule). Only spawn-loop boilerplate extracted.

5. **NotFound unification non-breaking.** Added `From` conversions + helper; 118 domain callers migrate organically.

6. **`RateLimitConfig::from_env()` not in `ServiceConfig`.** `ServiceConfig` is in `hkask-services-core` (no `hkask-cns` dep). Reading env vars at `ApiState` site matches existing pattern.

7. **G3 uses env-var file path, not DB.** Full per-user persistence needs schema design. Env-var approach makes boundary user-configurable now.
