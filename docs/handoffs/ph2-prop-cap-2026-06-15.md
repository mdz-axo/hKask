## 5) Key Decisions to Preserve

1. **Bash-only audit tooling (no Python).** The public seam inventory generator is a bash script, consistent with existing `check-unsafe.sh` and `check-unwrap-hotpaths.sh`. This avoids introducing a Python dependency into a pure Rust + bash project. Do not rewrite it in Python.

2. **Cross-crate name matching is scoped to same crate only.** The original v1 inventory matched item names across all crates (e.g., `fn new` in `hkask-mcp` matched a REQ test mentioning "new" in `hkask-agents`). Fixed by prefixing term data with `$cr:` and using `grep -qi "^${cr}:.*${name}"`. This reduced coverage from 49% (inflated) to 39% (accurate).

3. **Risk tier classification is heuristic, not semantic.** The `classify_risk()` function uses name patterns (e.g., `new`, `from_*`, `as_*`) and crate prefixes (e.g., `hkask-api` → route handler, `hkask-mcp-*` → tool handler). This is deliberately approximate — perfect classification would require understanding the codebase semantically. The heuristic is good enough for v1 triage.

4. **Proptest strategies use `prop_oneof!` with `Just(...)` for enum-like inputs.** Rather than implementing `Arbitrary` for `DelegationResource`/`DelegationAction`, the property tests use explicit strategy generators (`valid_resource_str()`, `valid_action_str()`). This keeps the test code self-contained and avoids adding `proptest` derive macros to production types.

5. **Property tests live in `mod proptest_tests` sub-modules within existing `mod tests`.** This separates property tests from unit tests for discoverability, while keeping them within `#[cfg(test)]` so they don't affect production builds. The existing unit tests (REQ types-event-001 through types-event-008) in `event.rs` are preserved as-is — property tests supplement, not replace.

6. **The "memory" → Registry alias in DelegationResource::parse_str is intentional.** The plan document and existing tests acknowledge this as a backward-compatibility feature. Property test `cap-prop-007` explicitly handles this case.

7. **CI inventory check uses `--check` mode (default) which strips dates before comparing.** The `GENERATED_DATE` placeholder is replaced with UTC timestamp on generation, then stripped before diffing. This prevents CI failures from timestamp drift while still detecting real content changes.
