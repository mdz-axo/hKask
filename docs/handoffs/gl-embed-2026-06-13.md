## 5. Key Decisions to Preserve

1. **Gentle dominates at 50%.** In an agent-native system, markdown specifications ARE the code. Stale documentation is a functional defect. Schriver 30% (findability), Hopper 10% (accessibility), Lovelace 10% (precision). Do not rebalance without understanding this rationale.

2. **All four exemplars are women — this is not incidental.** Technical documentation was founded (Hopper), algorithmized (Lovelace), measured (Schriver), and modernized (Gentle) by women. Preserve this credit in all manifests and templates.

3. **Per-dimension centroids, not just composite.** The diagnostic power is in per-dimension distances ("this doc is 0.3 from Hopper but 0.7 from Lovelace"). The composite is a summary; dimensions are the landscape.

4. **Classifier is config-driven, not hardcoded.** `registry/classify/{name}.yaml` files allow multiple classifiers to coexist (gemma, qwen, heuristic). corpus.yaml picks one. The Rust module is the execution engine; the YAML drives the WHAT.

5. **Corpus preparation is surgical.** Text was extracted, not edited. OCR artifacts are part of the authentic corpus. The centroid reflects actual source material.

6. **Per-type tag_weights deferred.** Failed essentialist G1 — global weights produce a valid composite. Per-type adjustments are a v2 refinement.

7. **DeepInfra + Gemma 4 26B is the classifier.** 4B active params, 95 req/s at 150 concurrent, ~$0.31 for all 38K passages, ~7 minutes runtime. Alternative classifiers can be added as new YAML files.

8. **The `document-update` skill is the integration point.** Gentle Lovelace feeds into Task 3 (Writing Quality Gate) as a 5th dimension. Don't create a separate evaluation path.
