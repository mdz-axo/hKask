## Key Decisions to Preserve

1. **Principle numbering is canonical**. `PRINCIPLES.md` and the spec's Principle Legend say: P1 = User Sovereignty, P2 = Affirmative Consent, P3 = Generative Space, P4 = Clear Boundaries, P5 = Essentialism, P7 = Evolutionary Architecture, P8 = Semantic Grounding, P9 = Homeostatic Self-Regulation, P12 = Subscriber Consent. Some earlier documents (including the prior handoff) incorrectly mapped P2 to User Sovereignty — use the canonical numbering.

2. **Motivating principle per domain**. A contract gets exactly one motivating principle in its ID prefix:
   - CNS regulation loops → P9
   - Wallet energy/balance → P9
   - Agents (consent) → P2; (sovereignty/pod) → P1; (ACP/adapters) → P4; (curator/loops/health) → P9
   - Storage (CRUD stores, embeddings, gallery, goals, triples, wallet_store) → P3; (user_store, sovereignty) → P1; (consent) → P2; (lock helpers, DB, path safety, contract tests) → P4; (spec types) → P8

3. **Constraining principles in body annotations**. Each production contract comment should carry `[P{N}] Motivating: …` and any relevant `[P{N}] Constraining: …` lines. Test/integration comments need only the new ID and description.

4. **The spec is the anchor**. After each crate, update `docs/architecture/core/FUNCTIONAL_SPECIFICATION.md` Section 3 (Non-CNS Domains) and Section 4.1 (Migration Summary) with the new IDs. Regenerate `docs/architecture/core/REQ_CONTRACT_INVENTORY.md`.

5. **Contract counts are discoverable, not fixed**. The handoff estimates (e.g., "storage: 12") were low; the actual source may contain far more IDs. Realign every `REQ:` occurrence rather than truncate to an estimate.

6. **Build must pass before finishing a crate**. Run `cargo check -p <crate>` and `bash scripts/gen-req-inventory.sh` as the final verification for every crate touched.
