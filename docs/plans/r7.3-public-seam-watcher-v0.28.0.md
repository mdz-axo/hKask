## 1) Objective

Make R7.3 (the CNS bot, owner of `hkask-cns`) the **watcher on the wall** for the public seam ŌĆö managing the inventory, monitoring coverage as a homeostatic signal, and reporting degradation through the existing CNS ŌåÆ Curator ŌåÆ CLI/REPL pathway.

Currently the public seam inventory is a CI-time static gate. It tells you *at build time* whether coverage drifted. It does not tell the *running system* that its behavioral guarantees are thinning. R7.3 closes that loop.

---

## 2) Why R7.3

| Candidate | Why Not |
|-----------|---------|
| R7.1 (storage) | Owns persistence, not observability |
| R7.2 (memory) | Owns semantic/episodic pipelines, not structural integrity |
| **R7.3 (cns)** | **Owns variety monitoring, algedonic alerting, homeostatic regulation. Public seam coverage IS a variety dimension.** |
| R7.4 (templates) | Owns patterns/registry ŌĆö could *hold* the inventory, but shouldn't *watch* it |
| R7.5 (agents) | Owns agent governance ŌĆö seam watching is infrastructure, not agent policy |
| R7.6 (mcp) | Owns tool dispatch ŌĆö wrong layer |
| R7.7 (cli) | Owns human interface ŌĆö reports, doesn't monitor |

R7.3 is the natural home because:

- **Conant-Ashby contract:** The CNS variety counter IS the regulator's model of system behavior. If the declared public API (the model) drifts from what's tested (the reality), that's a spec-drift signal ŌĆö exactly what `DefaultSpecCurator` already detects for specs.
- **Existing machinery:** `VarietyMonitor`, `AlgedonicManager`, `RuntimeAlert`, `CyberneticsLoop::sense/compute/act` ŌĆö all already exist. Adding a seam dimension is additive, not architectural.
- **Existing span:** `cns.architecture.module_depth` already tracks architectural depth as a CNS signal. Seam coverage is the same category ŌĆö architecture health.

## 3) What "Watching" Means ŌĆö Three Verbs

### Manage
Ingest the public seam inventory at startup. Hold it as a CNS-observable artifact. Detect when the inventory changes (code update ŌåÆ new inventory generated ŌåÆ drift from previous snapshot).

### Monitor
Track per-crate coverage as a variety dimension. The "states" are individual public items; the "variety" is the count of covered items. Deficit = drop in covered items since last snapshot. This is a **positive variety** signal ŌĆö we want coverage to go *up*, not down.

### Report
Surface degradation through the existing algedonic pathway:
```
Seam coverage drop ŌåÆ VarietyMonitor deficit ŌåÆ AlgedonicManager.check()
    ŌåÆ RuntimeAlert (domain="seam:{crate}") ŌåÆ CyberneticsLoop::sense()
    ŌåÆ SignalMetric::SeamCoverage ŌåÆ Deviation ŌåÆ Escalate(Curation)
    ŌåÆ Curator surfaces in CLI/REPL dual-presence
```

Positive signals (coverage improvements) are also reported ŌĆö not as alerts, but as CNS spans that the Curator can surface as "system health improving."

### Wave 3 ŌĆö Seam watcher module in hkask-cns

**What:** New module that loads the inventory, registers variety domains, and emits CNS spans on change.

**New file:** `hkask-cns/src/seam_watcher.rs`

**Public API (Ōēż5 public items ŌĆö deep module discipline):**

```rust
/// Watches the public seam ŌĆö loads inventory, tracks coverage, emits CNS spans.
pub struct SeamWatcher {
    /// Current inventory snapshot
    inventory: SeamInventory,
    /// Previous snapshot for drift detection (None on first load)
    previous: Option<SeamInventory>,
    /// Path to the JSON inventory file
    inventory_path: PathBuf,
}

impl SeamWatcher {
    /// Load inventory from the JSON file at `path`.
    /// Returns None if the file doesn't exist (non-fatal ŌĆö system runs without seam watching).
    pub fn load(path: impl Into<PathBuf>) -> Option<Self>;

    /// Register all per-crate seam domains in the VarietyMonitor.
    /// Expected variety = covered_items (we want this to go up).
    /// Called once at startup after load.
    pub fn register_domains(&self, runtime: &CnsRuntime);

    /// Check for drift: compare current inventory against previous snapshot.
    /// Returns a list of per-crate coverage deltas.
    /// Positive delta = coverage improved. Negative delta = coverage degraded.
    pub fn check_drift(&mut self) -> Vec<SeamDrift>;

    /// Emit CNS spans for the current coverage state.
    /// - `cns.architecture.seam.coverage` ŌĆö per-crate coverage ratio
    /// - `cns.architecture.seam.drift` ŌĆö per-crate delta since last check
    pub async fn emit_spans(&self, sink: &NuEventSink);

    /// Update the inventory (e.g., after a code update regenerates the JSON).
    /// Moves current ŌåÆ previous, loads new from disk.
    pub fn refresh(&mut self) -> Option<SeamInventory>;
}

/// Per-crate coverage delta since last snapshot.
pub struct SeamDrift {
    pub crate_name: String,
    pub previous_coverage_pct: f64,
    pub current_coverage_pct: f64,
    pub delta_pct: f64,        // positive = improvement
    pub items_added: i64,      // new public items (can be negative if removed)
    pub items_covered_delta: i64, // change in covered count
}
```

**Integration point:** `CnsRuntime` gets a new method:
```rust
impl CnsRuntime {
    /// Initialize seam watching. Loads inventory, registers domains.
    /// Non-fatal: if the JSON file is missing, seam watching is disabled.
    pub async fn init_seam_watcher(&self, inventory_path: &str) -> Option<SeamWatcher>;
}
```

**Verification:** `cargo test -p hkask-cns` ŌĆö new tests for:
- `SeamWatcher::load` parses valid JSON
- `SeamWatcher::load` returns None for missing file (non-fatal)
- `check_drift` detects coverage drop (negative delta)
- `check_drift` detects coverage improvement (positive delta)
- `check_drift` returns empty vec when no change
- `register_domains` sets expected variety = covered_items per crate

## 5) What This Does NOT Do

| Non-goal | Why |
|----------|-----|
| Runtime API call interception | Compiler already enforces `pub` boundaries at build time. OCAP enforces capability boundaries at runtime. Duplicating the compiler's work adds latency for zero gain. |
| Blocking builds on low coverage | CI already fails on inventory drift. CNS observes and alerts ŌĆö it does not block. |
| Auto-generating tests | R7.3 watches, reports, escalates. It does not write code. That's the Curator's metacognitive domain. |
| New MCP server | Seam watching is internal CNS infrastructure, not a tool surface. No `hkask-mcp-seam` crate. |
| New 7R7 bot | R7.3 already exists. Adding a bot for one signal violates P5 (Essentialism). |
| Per-item runtime tracking | We track aggregate coverage per crate, not individual `pub fn` call counts. The inventory is a snapshot, not a live trace. |

| # | Gap | Severity | Phase |
|---|-----|----------|-------|
| G1 | **Missing afferent signal** ŌĆö nothing calls `increment_variety()` for seam domains. The alert pathway is wired but never triggered. `CyberneticsLoop::sense()` reads from `VarietyMonitor`, but seam domains receive no increments. | ­¤ö┤ Critical | Phase 2 (Cybernetic) |

| Gap | Resolution | Wave |
|-----|-----------|------|
| G1 | `SeamWatcher::check_drift()` calls `cns_runtime.increment_variety()` for each crate domain with current `covered_items` count. This closes the afferent path: drift check ŌåÆ variety increment ŌåÆ algedonic check ŌåÆ alert. | Wave 3 (revised) |
| G2 | Embed the inventory JSON at compile time via `include_str!()` in a new `hkask-cns/src/seam_inventory_embedded.rs` that wraps the embedded bytes. | G3 | Consolidate to Ōēż7 public items: merge `emit_spans()` into `check_drift()` (drift checking naturally produces spans). Replace `inventory()`/`overall_coverage()`/`crate_coverage()` with a single `summary() -> SeamSummary` method. Final surface: `SeamDrift`, `SeamSummary`, `SeamWatcher`, `load()`, `register_domains()`, `check_drift()`, `refresh()`. | Wave 3 (revised) |
| G4 | Document that seam domains use **snapshot comparison**, not event streaming. The `VarietyTracker` window is bypassed: `check_drift()` directly calls `increment_variety()` with the current count, and `check_variety()` immediately after. | G5 | Add `AboveSetPoint` handling for `SeamCoverage` in `compute()`: emit an info-level `LoopAction::new(LoopId::Curation, ActionType::Notify, ...)` with `reason: "seam_coverage_improved"`. The Curator surfaces this as a positive health signal. | Wave 4 (revised) |

### Design Decision: CNS Integration Justified

The pragmatics Phase 3 (laziness) considered a simpler alternative: Curator reads
`inventory.json` directly on `/status`, no CNS integration. This would be ~50 lines
vs. ~650. **Rejected** because:
- Passive alerting (system tells you) requires less human attention than active
  polling (you check `/status`). Same pattern as wallet balance alerts.
- CNS integration provides persistent event log (NuEventSink) for audit trail.
- Variety tracking enables per-crate threshold tuning via set-points.
- The brachistochrone principle: the longer path (CNS integration) reduces total
  system action by automating the trigger.

## 6) Acceptance Criteria

1. **JSON inventory** generated alongside markdown, CI-enforced to stay in sync.
2. **CNS spans** `cns.architecture.seam.coverage` and `cns.architecture.seam.drift` are valid canonical namespaces.
3. **SeamWatcher** loads inventory at startup (embedded JSON, file fallback), registers per-crate variety domains.
4. **Afferent signal closed** ŌĆö `check_drift()` increments variety for each crate domain, triggering algedonic check.
5. **Coverage drop** in any crate fires a `RuntimeAlert` through the existing algedonic pathway.
6. **Coverage improvement** emits an info-level CNS span via `AboveSetPoint` handling (no alert ŌĆö positive signal).
7. **Curator `/status`** shows seam coverage alongside other variety metrics.
8. **R7.3 domains** include `"seam"` ŌĆö the bot knows it owns this concern.
9. **Non-fatal:** missing inventory (no embedded JSON, no file) disables seam watching silently.
10. **Deployment-safe:** embedded JSON works in `cargo install` binaries. File path is a development override.
11. **Deep module:** Ōēż7 public items on `SeamWatcher`.
12. **Zero new dependencies.** JSON parsing uses `serde_json` (already present).

## 7) Risk Assessment

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Inventory JSON missing at runtime | Medium (first run, no CI) | Non-fatal ŌĆö seam watching disabled, system runs normally |
| Coverage alert fatigue (39% baseline means constant warnings) | High | Set-point is *regression from previous snapshot*, not absolute threshold. System alerts only when coverage *drops*, not when it's low. |
| JSON/Markdown drift | Low | CI gate ensures both outputs stay in sync |
| Performance impact | Negligible | Inventory loaded once at startup (~25 crates ├Ś ~100 items = ~2,500 entries). Drift check is O(n) comparison. No per-call overhead. |
