# Semantic Graph Audit

Domain-agnostic semantic dependency graph analysis. Accepts any directed graph (code modules, crates, skills, ADRs, CNS spans, decision trees, data flows), classifies edges by constraint force, detects cycles/redundancies/ gaps/orphans, and evaluates graph health through pragmatic-semantics, pragmatic-cybernetics, essentialist, and grill-me lenses.


## When to Use

- When you need to analyze a directed dependency graph of any domain — code modules, crates, skills, ADRs, CNS spans, decision trees, or data flows.
- When you need to classify graph edges by constraint force to understand which dependencies are hard, soft, or advisory.
- When you need to detect structural pathologies: cycles, redundancies, gaps, or orphaned nodes.
- When you need to evaluate overall graph health through pragmatic-semantics, pragmatic-cybernetics, essentialist, or grill-me analytical lenses.
