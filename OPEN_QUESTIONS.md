## 2. Open-Loop Constraints (TASK 1)

### 2.1 OCAP Enforcement

**Finding:** OCAP capability checks are an open-loop Prohibition (no feedback path). When a token is denied, the system does not adapt — denial is terminal.

**Analysis:** Per PRINCIPLES.md P4 (Clear Boundaries), OCAP gates are intentionally non-cybernetic. They are Prohibitions (affirmative consent boundaries), not Guardrails (adjustable thresholds). The lack of feedback is by-design, not a defect.

**Question:** Should OCAP remain a pure Prohibition gate, or should the Curator receive capability-denial signals for adaptive policy recommendations?

**Proposed resolution:** Keep as-is. Document in PRINCIPLES.md §2.1 P4 that "OCAP gates are Prohibitions, not regulatory loops" to prevent future confusion.

**Status:** Document (no code change required)

### 2.2 Consent Enforcement

**Finding:** Same open-loop pattern as OCAP — ConsentManager denies silently.

**Question:** Should consent denials produce CNS events for observability?

**Proposed resolution:** Add `cns.consent.denied` ν-event emission from `ConsentManager::has_consent()` when denial occurs. This provides observability without changing the Prohibition enforcement model. The event is OBSERVE-only; it does not open a feedback path.

**Status:** OPEN — requires instrumentation in `hkask-agents::consent`

### 2.3 SpecDriftAlert

**Finding:** `DefaultSpecCurator` detects spec-implementation divergence but only reports it. No actuator exists for automated resolution.

**Question:** Should spec drift auto-propose revisions, or remain advisory-only?

**Proposed resolution:** Keep advisory. Specs represent user intent (MDS §2: Curation decisions are made by Curator or human). Automated revision would violate user sovereignty (Magna Carta P1). Add `LoopPayload::SpecDriftResolved` for when a human resolves the drift, closing the observability loop.

**Status:** OPEN — requires `SpecDriftResolved` payload type in `hkask-types::loops`

**Status:** OPEN — requires condenser instrumentation and SLA definition
